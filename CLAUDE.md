# 고구마마켓 프로젝트

개발 연습용 프로젝트입니다. Next.js(화면 구성) + Supabase(데이터베이스) + Vercel(웹에 올리기) 조합으로 개발합니다.

---

## 소통 방식

- 개발 용어는 비개발자도 이해할 수 있게 쉽게 풀어서 설명한다.
  - 예: "컴포넌트" → "화면 조각", "API" → "데이터를 주고받는 통로", "마이그레이션" → "데이터베이스 구조 변경"
- 지금 무엇을 하고 있는지 작업 단계마다 한 문장으로 말해준다.
- 작업이 길어질 경우 중간중간 진행 상황을 알려준다.

---

## 작업 원칙

- 사용자가 지시한 것만 단계적으로 수행한다. 요청하지 않은 기능 추가나 리팩터링은 하지 않는다.
- 자동으로 할 수 있는 것은 최대한 직접 처리한다.
- 꼭 사람이 직접 해야 하는 것(예: Vercel 배포 버튼 클릭, Supabase 대시보드 설정)만 쉬운 말로 단계별로 안내한다.
- 작업이 끝나면 항상 결과를 바로 브라우저에서 확인할 수 있도록 앱을 실행하거나 확인 방법을 안내한다.

---

## 기술 스택 설명 (쉬운 버전)

| 도구 | 쉬운 설명 |
|------|-----------|
| Next.js | 웹사이트 화면을 만드는 틀 |
| Supabase | 데이터를 저장하고 불러오는 창고 + 로그인 기능 |
| Vercel | 만든 사이트를 인터넷에 올려주는 서비스 |

---

## 개발 전 확인사항

Next.js는 버전마다 문법이 크게 다를 수 있으므로, 코드를 작성하기 전에 반드시 `node_modules/next/dist/docs/` 안의 가이드를 먼저 읽는다.
